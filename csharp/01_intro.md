## Station 1: 環境構築と Hello World

### クリア条件
- .NET SDK がインストールされ、`dotnet --version` でバージョンが表示されること
- `dotnet new console` でプロジェクトを作成し、`dotnet run` で "Hello, World!" が表示されること
- 作成したプロジェクトを GitHub にプッシュできていること

### 問題概要
C# の開発に必要な .NET SDK と開発環境（Visual Studio または VS Code）をセットアップする。最初のプログラムとして "Hello, World!" を動かす。

**ツールの役割**
| ツール | 役割 |
|---|---|
| .NET SDK | C# のコンパイラ・実行ランタイムを含む開発キット |
| `dotnet` コマンド | プロジェクト作成・ビルド・実行をターミナルから行うツール |
| Visual Studio / VS Code | コードを書くためのエディタ |

### 確認テスト
1. .NET 8 SDK（以上）をインストールする
2. ターミナルで `dotnet --version` を実行し、バージョンが表示されることを確認する
3. `dotnet new console -n HelloWorld` でプロジェクトを作成する
4. `Program.cs` を開いて `Console.WriteLine("Hello, World!");` が書かれていることを確認する
5. `dotnet run` を実行して出力を確認する
6. 出力メッセージを自分の名前に変更して再度実行する
7. GitHub にリポジトリを作成してプッシュする

### 参考資料
- [.NET ダウンロード](https://dotnet.microsoft.com/download)
- [C# の最初のステップ - Microsoft Learn](https://learn.microsoft.com/ja-jp/dotnet/csharp/tour-of-csharp/)

## Station 2: 変数とデータ型

### クリア条件
- `int`, `double`, `string`, `bool` の 4 つの型を使った変数宣言ができること
- `var` を使った型推論の変数宣言ができること
- `Console.ReadLine()` でユーザーからの入力を受け取り、型変換して計算に使えること

### 問題概要
C# は「静的型付け言語」であり、変数には必ず型がある。基本的なデータ型を理解し、ユーザー入力を受け取るプログラムを作る。

**主要なデータ型**
| 型 | 説明 | 例 |
|---|---|---|
| `int` | 整数 | `42` |
| `double` | 小数 | `3.14` |
| `string` | 文字列 | `"こんにちは"` |
| `bool` | 真偽値 | `true` / `false` |
| `decimal` | 高精度小数（金額向き） | `1234.56m` |

### 確認テスト
1. `int`, `double`, `string`, `bool` それぞれの型で変数を宣言し、`Console.WriteLine` で出力する
2. `var` を使って型推論で変数を宣言する
3. 文字列補間（`$"名前は{name}です"`）を使って出力する
4. `Console.ReadLine()` でユーザーの名前を入力してもらい、挨拶を表示するプログラムを作る
5. `Console.ReadLine()` で数値を 2 つ入力してもらい、その合計を表示するプログラムを作る
   - ヒント: `int.Parse()` または `int.TryParse()` で文字列を数値に変換する

### 参考資料
- [C# の型とメンバー](https://learn.microsoft.com/ja-jp/dotnet/csharp/tour-of-csharp/#types-and-members)

## Station 3: 条件分岐

### クリア条件
- `if` / `else if` / `else` を使った条件分岐が書けること
- `switch` 文を使った条件分岐が書けること
- 比較演算子（`==`, `!=`, `<`, `>`, `<=`, `>=`）と論理演算子（`&&`, `||`, `!`）を使えること

### 問題概要
プログラムの流れを条件によって変える「条件分岐」を学ぶ。ユーザーの入力に応じて異なる処理を実行するプログラムを作る。

### 確認テスト
1. ユーザーに年齢を入力してもらい、以下の条件で異なるメッセージを表示する
   - 18 歳未満: 「未成年です」
   - 18 歳以上 65 歳未満: 「成人です」
   - 65 歳以上: 「シニアです」
2. ユーザーに曜日を入力してもらい、`switch` 文で以下を表示する
   - 月〜金: 「平日です」
   - 土〜日: 「休日です」
   - それ以外: 「正しい曜日を入力してください」
3. ユーザーに点数（0〜100）を入力してもらい、成績（A〜F）を表示するプログラムを作る

### 参考資料
- [C# の if 文](https://learn.microsoft.com/ja-jp/dotnet/csharp/language-reference/statements/selection-statements#the-if-statement)
- [C# の switch 文](https://learn.microsoft.com/ja-jp/dotnet/csharp/language-reference/statements/selection-statements#the-switch-statement)

## Station 4: ループ（繰り返し処理）

### クリア条件
- `for` ループを使って指定回数の繰り返しができること
- `while` ループを使って条件に基づく繰り返しができること
- `foreach` ループを使って配列の要素を 1 つずつ処理できること
- `break` と `continue` を使ってループの制御ができること

### 問題概要
同じ処理を繰り返し実行する「ループ」を学ぶ。配列と組み合わせて、複数のデータを処理するプログラムを作る。

**ループの種類**
| ループ | 用途 |
|---|---|
| `for` | 回数が決まっている繰り返し |
| `while` | 条件が満たされている間、繰り返す |
| `do-while` | 最低 1 回は実行し、その後条件を確認 |
| `foreach` | 配列やリストの全要素を順に処理 |

### 確認テスト
1. `for` ループで 1 から 10 までの合計を計算して表示する
2. `for` ループで九九の表を表示する
3. `while` ループで、ユーザーが "exit" と入力するまで繰り返し入力を受け付けるプログラムを作る
4. `string[]` 配列にいくつかの名前を入れ、`foreach` で全員に挨拶を表示する
5. 1 から 100 までの数値のうち、3 の倍数だけを表示するプログラムを `for` + `continue` で作る

### 参考資料
- [C# の反復ステートメント](https://learn.microsoft.com/ja-jp/dotnet/csharp/language-reference/statements/iteration-statements)

## Station 5: メソッド（関数）

### クリア条件
- 引数と戻り値のあるメソッドを定義して呼び出せること
- `void` メソッド（戻り値なし）を定義して呼び出せること
- メソッドのオーバーロード（同名で引数違い）を定義できること
- 省略可能引数（デフォルト値付き引数）を使えること

### 問題概要
コードを意味のある単位にまとめる「メソッド」を学ぶ。メソッドを使うことでコードの再利用性と可読性が向上する。

### 確認テスト
1. 2 つの整数を受け取り、合計を返す `Add(int a, int b)` メソッドを作成する
2. 名前を受け取って挨拶を表示する `Greet(string name)` メソッド（戻り値 `void`）を作成する
3. `Add` メソッドをオーバーロードして、3 つの整数を受け取るバージョンも作る
4. 消費税を計算する `CalcTax(int price, double rate = 0.10)` メソッドを作成する
   - `rate` を省略した場合は 10% で計算する
5. これまでのプログラムからメソッドとして切り出せる処理を見つけ、リファクタリングする

### 参考資料
- [C# のメソッド](https://learn.microsoft.com/ja-jp/dotnet/csharp/methods)

## Station 6: クラスとオブジェクト

### クリア条件
- クラスを定義し、プロパティとメソッドを持たせられること
- `new` キーワードでインスタンスを作成し、メソッドを呼び出せること
- コンストラクタを定義し、初期化処理を書けること
- `public` / `private` のアクセス修飾子を理解し使い分けられること

### 問題概要
C# はオブジェクト指向言語。データと処理をまとめた「クラス」を定義し、それをもとに「オブジェクト（インスタンス）」を作る考え方を学ぶ。

**クラスの構成要素**
| 要素 | 説明 |
|---|---|
| プロパティ | データを保持する（例: `Name`, `Price`） |
| メソッド | 処理を定義する（例: `CalcTotal()`） |
| コンストラクタ | インスタンス作成時の初期化処理 |
| アクセス修飾子 | 外部から見えるか（`public`）、隠すか（`private`） |

### 確認テスト
1. `Item` クラスを作成する
   - プロパティ: `Name`（string）、`Price`（int）
   - メソッド: `GetInfo()` → 「〇〇は△△円です」を返す
2. コンストラクタで `Name` と `Price` を受け取るようにする
3. `Main` メソッドから `Item` のインスタンスを複数作成し、`GetInfo()` を呼び出す
4. `Price` を `private set;` にして、外部からの直接変更を防ぐ
5. ファイルを分離する: `Item.cs` を別ファイルに切り出す

### 参考資料
- [C# のクラス](https://learn.microsoft.com/ja-jp/dotnet/csharp/fundamentals/types/classes)

## Station 7: 配列とリスト

### クリア条件
- 配列（`int[]`）を使ってデータを格納・取り出しできること
- `List<T>` を使って動的にデータの追加・削除ができること
- `Dictionary<TKey, TValue>` を使ってキーと値のペアでデータを管理できること

### 問題概要
複数のデータをまとめて管理するための「コレクション」を学ぶ。配列は固定長で基本を学び、実務では可変長の `List<T>` を使うことが多い。

**主なコレクション**
| 型 | 特徴 |
|---|---|
| `int[]` | 固定長の配列。サイズ変更不可 |
| `List<T>` | 可変長のリスト。追加・削除が自由 |
| `Dictionary<TKey, TValue>` | キーと値のペアで管理 |

### 確認テスト
1. `int[]` 配列に 5 つの数値を入れ、合計と平均を計算する
2. `List<string>` で買い物リストを作り、以下の操作を行う
   - `Add()` で商品を追加
   - `Remove()` で商品を削除
   - `Contains()` で商品の有無を確認
   - `Count` で件数を表示
   - `foreach` で全件表示
3. `Dictionary<string, int>` でフルーツの名前と価格を管理し、以下を行う
   - 要素の追加・取得・削除
   - `ContainsKey()` でキーの存在確認

### 参考資料
- [C# のコレクション](https://learn.microsoft.com/ja-jp/dotnet/csharp/programming-guide/concepts/collections)

## Station 8: 例外処理

### クリア条件
- `try-catch` で例外をキャッチし、エラーメッセージを表示できること
- `finally` ブロックの役割を理解していること
- 特定の例外（`FormatException`, `ArgumentException` など）を個別にキャッチできること
- 自分で例外を `throw` できること

### 問題概要
プログラムの実行中に起きるエラー（例外）を適切に処理する方法を学ぶ。例外処理をしないとプログラムがクラッシュするが、適切に処理すればユーザーにわかりやすいメッセージを出して処理を続行できる。

### 確認テスト
1. ユーザーから数値の入力を受け取り、`try-catch` で数値以外が入力された場合にエラーメッセージを表示する
2. 0 で割り算した場合の `DivideByZeroException` をキャッチする
3. `finally` ブロックを使い、成功・失敗にかかわらずメッセージを表示する
4. マイナスの金額が入力された場合に `ArgumentException` を `throw` するメソッドを作る

### 参考資料
- [C# の例外処理](https://learn.microsoft.com/ja-jp/dotnet/csharp/fundamentals/exceptions/)

## Station 9: ミニ家計簿アプリの作成（総合演習）

### クリア条件
- コンソールアプリとして動作する「ミニ家計簿」が完成していること
- Station 1〜8 で学んだ知識がすべて使われていること
- GitHub にプッシュされていること

### 問題概要
これまで学んだ知識を総動員して、コンソールで動く「ミニ家計簿アプリ」を完成させる。

**必要な機能**
| 機能 | 使う知識 |
|---|---|
| メニュー表示・選択 | ループ・条件分岐 |
| 収入/支出の登録 | クラス・`List<T>` |
| 一覧表示 | `foreach`・文字列補間 |
| カテゴリ別集計 | `Dictionary<TKey, TValue>` |
| 合計・残高の計算 | メソッド |
| 入力エラーへの対応 | 例外処理 |

### 確認テスト
1. 起動するとメニューが表示される（1. 登録 / 2. 一覧 / 3. 集計 / 4. 終了）
2. `Entry` クラスを作成する（プロパティ: `Date`, `Category`, `Amount`, `IsIncome`）
3. 収入・支出を登録できる
   - 日付・カテゴリ・金額を入力して `List<Entry>` に追加する
   - 不正な入力はエラーメッセージを出して再入力させる
4. 登録した一覧を表示できる
5. カテゴリ別の合計を表示できる
6. 総収入・総支出・残高を表示できる
7. "終了" を選ぶまでメニューがループする
8. GitHub にプッシュする
